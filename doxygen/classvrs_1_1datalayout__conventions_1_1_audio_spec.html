Detailed Description

DataLayout definitions use to describe what's in an audio content block.

These names and types are a convention that enables us to find audio block spec within a DataLayout block, which is either before the audio content block in the same record, or in the last Configuration record. Note that once a configuration location is found, the next time around, the same location will be used again. AudioFormat is optional for legacy reasons, and is assumed to be AudioFormat::PCM when missing.

Note that the values used are not static, so that if the configuration changes, the latest value is used, without having to search each time we have a new image block.
